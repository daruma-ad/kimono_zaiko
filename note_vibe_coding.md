### この記事で完成する「スマホQR棚卸システム」でできること

特別な機器の購入は一切不要。お手持ちのスマートフォンと、無料のGoogleスプレッドシートだけで動きます。

1. **スマホカメラで高速スキャン**: 
   スマホの背面カメラで値札のQRコードをかざすだけで、瞬時に読み取ります。
2. **ピピッ！と鳴る業務用の操作感**: 
   読み取りに成功すると、画面が一瞬緑色に光り、**「ピピピッ」と電子音**が鳴ります。まるで数十万円の業務用端末を操作しているような快適さです。
3. **Googleスプレッドシートにリアルタイム自動記録**: 
   スマホでスキャンした瞬間、自動的にスプレッドシートの「最終棚卸日」が現在時刻に書き換わります。PC側への手入力は一切不要です。
4. **値札用QRタグの自動作成＆印刷**: 
   新しい商品を入荷したら、アプリ画面で商品名や価格を入力するだけで、その場でQRコード付きの「管理タグ（値札）」が生成され、ワンタップでプリンターやラベル用紙に印刷できます。
5. **棚卸し漏れが一目でわかる在庫一覧**: 
   登録した商品はスマホの画面に一覧表示され、棚卸しが「完了したものは緑色」「まだのものは赤色」と自動で色分けされます。キーワード検索や「未完了のみ表示」といった絞り込みも一瞬です。

### 💡 AIとの「会話（Vibe）」を進めるコツ

AIからコードが出力された後、実際に触ってみて「ここをもっとこうしたい！」と思ったら、以下のようにおしゃべりするように指示を追加してください。

* **「もっと項目を増やしたい」**
  👉 *「登録フォームに、『仕入先』と『担当者名』の入力欄も追加して、スプレッドシートにも保存できるようにコードを修正して。」*
* **「音や見た目を変えたい」**
  👉 *「スキャンしたときの『ピピッ』という音をもっと高い音にして、フラッシュの色を黄色に変えてほしい。」*
* **「写真を登録できるようにしたい」**
  👉 *「スマホのカメラで商品の写真を撮って、その画像もスプレッドシート（またはGoogleドライブ）に保存できる仕組みに拡張したい。どうすればいい？」*

このように、日本語でリクエストを投げるだけで、AIが自動的にプログラムを書き換えてくれます。これがVibe Codingの強みです。

## 第2章：AIが作ったアプリをネットで公開する手順

AIがプログラム（ファイル）を出力してくれたら、次はそれを「設定」してネット上に公開する作業です。専門知識がなくても、以下の**5つのステップ**を順番に進めれば、スマホで動くようになります。

### 【ステップ1】Googleスプレッドシートの作成
1. Googleドライブを開き、新規で **「Google スプレッドシート」** を作成します。
2. スプレッドシートの名前を「在庫管理」などに変更し、シートの名前を **`シート1`** に設定します。
3. シートの1行目（A1〜J1のセル）に、以下の英単語を左から順番に正確に入力します。
   - `code`（商品番号が入る列）
   - `name`（商品名が入る列）
   - `type`（種類が入る列）
   - `tailor`（仕立て状態が入る列）
   - `priceSale`（販売価格が入る列）
   - `priceRent`（レンタル価格が入る列）
   - `sizeMitake`（身丈が入る列）
   - `sizeYuki`（裄丈が入る列）
   - `status`（ステータスが入る列）
   - `lastAuditDate`（最終棚卸日が入る列）

---

### 【ステップ2】GAS（橋渡しプログラム）の設定と公開
1. スプレッドシートのメニューから **「拡張機能」 ＞ 「Apps Script」** をクリックします。
2. もともと書かれている文字をすべて削除し、AIが出力した **`コード.gs`** をコピーして貼り付けます。
3. 上部の **「保存（フロッピーディスクのアイコン）」** をクリックします。
4. 右上の青い **「デプロイ」ボタン ＞ 「新しいデプロイ」** を選択します。
5. 歯車マークから「ウェブアプリ」を選択し、以下を設定します。
   - **説明**: `在庫管理アプリAPI`
   - **次のユーザーとして実行**: `自分 (あなたのGoogleアカウント)`
   - **アクセスできるユーザー**: ⚠️必ず **`全員` (Anyone)** を選択してください。
6. 青い「デプロイ」をクリックし、Googleの承認警告が出たら、**「Advanced」 ＞ 「Go to ... (unsafe)」 ＞ 「Allow（許可）」** の順にクリックします。
7. 画面に表示される **「ウェブアプリのURL」**（`/exec`で終わるURL）をコピーしてメモしておきます。

---

### 【ステップ3】アプリのURLを設定し、GitHubへアップロード
1. AIが作成したファイルの中の **`api.js`** というファイルをメモ帳などで開きます。
2. `GAS_API_URL` という設定箇所を見つけ、そこにステップ2でコピーした「ウェブアプリのURL」を貼り付けて保存します。
   ```javascript
   const USE_GAS_API = true; // ここを true にする
   const GAS_API_URL = "【ここにコピーしたURLを貼り付け】";
   ```
3. 無料のファイル置き場である **[GitHub](https://github.com/)** にログインし、新しいリポジトリ（フォルダ）を作成します。
4. 作成したリポジトリに、AIが作成したファイル群（`index.html`、`style.css`、`api.js`、`app.js` など）をドラッグ＆ドロップでまとめてアップロードします。

---

### 【ステップ4】GitHub PagesでWebアプリを公開
1. アップロードしたGitHubの画面で、右上の **「Settings」** タブを開きます。
2. 左メニューから **「Pages」** をクリックします。
3. **Build and deployment** の中にある **Source** の設定を、**`GitHub Actions`** に切り替えます。
4. 1分ほど待つと、画面上部にあなたのアプリ専用のURL（例: `https://<アカウント名>.github.io/kimono_zaiko/`）が発行されます。

---

### 【ステップ5】スマートフォンでテスト起動！
1. 発行されたURLに、お手持ちのスマートフォンでアクセスします。
2. 初回起動時、カメラの使用許可を求められますので「許可」します。
3. 「新規登録」タブからテスト商品を登録すると、自動で値札QRコードが生成され、同時に**Googleスプレッドシートの2行目にデータが瞬時に追加**されます！
4. 登録したQRコードをスマホカメラで読み取って、シートの「最終棚卸日」が自動更新されるか確認してください。

## 第3章：動かないときの「トラブル解消チェックリスト」

Vibe Codingで最もつまずきやすいポイントと、その簡単な解決策です。

### Q. 「ファイルの読み込みに失敗しました」と表示される / スプレッドシートに反映されない
- **原因1: スプレッドシートの1行目の英語名が違っている**
  `code` や `lastAuditDate` などのスペルや大文字小文字が1文字でも違っていると、GASが混乱してエラーになります。ステップ1の見出しを確認してください。
- **原因2: Googleの「複数ログイン制限」にかかっている**
  ブラウザで複数のGoogleアカウントにログインしている場合、Googleのセキュリティが通信をブロックします。
  👉 **対策**: ブラウザの「シークレットモード（プライベートブラウズ）」でアプリを開いてみてください。これで正常に動けば、連携自体は成功しています。
